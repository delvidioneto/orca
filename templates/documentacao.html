Documentação

Como criar pipelines e tarefas e configurar o agendamento via JSON.

1. O que é um Pipeline

Um pipeline é um conjunto de tarefas que podem ser executadas em sequência ou com dependências. Você cria primeiro o pipeline (nome e descrição) e depois adiciona as tarefas a ele. Cada tarefa pode rodar um script (Python, Batch ou PowerShell) em um horário ou intervalo definido por um JSON.

2. Como criar um Pipeline

  1. No menu, clique em Pipelines.
  2. Clique no botão Novo Pipeline (ou equivalente).
  3. Preencha:
    • Nome — nome do pipeline (ex.: "Backup diário").
    • Descrição — opcional; texto livre.
    • Ativo — marque para o pipeline ser considerado pelo agendador; desmarque para pausar todas as tarefas.
  4. Clique em Salvar.
  5. Você será redirecionado para a lista de pipelines; clique no nome do pipeline criado para abrir o detalhe e adicionar tarefas.

3. Como criar Tarefas

  1. Abra o Pipeline (clique no nome na lista de pipelines).
  2. Na tela de detalhe do pipeline, clique em Nova Tarefa (ou no link/botão equivalente que leva à criação de tarefa com esse pipeline já selecionado).
  3. Preencha o formulário:
    • Pipeline — selecione o pipeline ao qual a tarefa pertence (pode já vir preenchido se você veio do detalhe do pipeline).
    • Nome — nome da tarefa (ex.: "Backup do banco").
    • Descrição — opcional.
    • Caminho do Script — caminho relativo ao diretório do projeto, ex.: scripts/meu_script.py ou scripts/backup.bat.
    • Tipo — Python (.py), Shell (.sh), Batch (.bat), PowerShell (.ps1), Node.js (.js), Perl (.pl), Ruby (.rb) ou Go (.go). O tipo pode ser inferido pela extensão do arquivo. Cada tipo só roda no SO compatível (ex.: .bat só no Windows; .sh só em Linux/macOS).
    • Depende de — opcional; selecione outras tarefas do mesmo pipeline que precisam ter rodado com sucesso antes desta (segure Ctrl para múltiplas).
    • Tentativas — quantas vezes tentar de novo em caso de falha (0 = não tentar).
    • Delay entre tentativas (s) — segundos de espera entre uma tentativa e outra.
    • Timeout (s) — opcional; tempo máximo de execução em segundos; em branco = sem limite.
    • Configuração do Agendamento (JSON) — objeto JSON que define quando a tarefa roda (veja as tabelas abaixo). Pode ser {} para diário à meia-noite.
    • Tarefa ativa — marque para a tarefa ser agendada; desmarque para desativar.
  4. Clique em Salvar.

O tipo de agendamento (intervalo, diário, semanal, etc.) é inferido automaticamente a partir das chaves do JSON; não há campo separado para isso.

4. Configuração do Agendamento (JSON)

O campo Configuração do Agendamento é um único objeto JSON. O sistema define o tipo de agendamento pelas chaves presentes, conforme a tabela de inferência abaixo. Em seguida, para cada tipo, as chaves aceitas estão nas tabelas por tipo.

4.1 Inferência do tipo (qual JSON gera qual agendamento)

Se o JSON contiver... Tipo inferido
seconds, minutes, hours ou daysIntervalo
weekQuinzenal
day (dia do mês, 1–31)Mensal
day_of_week (0–6)Semanal
year, month ou secondCron
Nenhuma das acima (ou só hour/minute)Diário
Vazio {}Diário (meia-noite)

4.3 Diário (um horário por dia)
