Pipelines e Tarefas

Como criar pipelines, tarefas e configurar o agendamento via JSON.

1. O que é um Pipeline

Um pipeline é um conjunto de tarefas que podem ser executadas em sequência ou com dependências. Você cria primeiro o pipeline (nome e descrição) e depois adiciona as tarefas a ele. Cada tarefa pode rodar um script (Python, Batch ou PowerShell) em um horário ou intervalo definido por um JSON.

2. Como criar um Pipeline

  1. No menu, clique em Pipelines.
  2. Clique no botão Novo Pipeline (ou equivalente).
  3. Preencha:
    • Nome — nome do pipeline (ex.: "Backup diário").
    • Descrição — opcional; texto livre.
    • Ativo — marque para o pipeline ser considerado pelo agendador; desmarque para pausar todas as tarefas.
  4. Clique em Salvar.
  5. Você será redirecionado para a lista de pipelines; clique no nome do pipeline criado para abrir o detalhe e adicionar tarefas.

3. Como criar Tarefas

  1. Abra o Pipeline (clique no nome na lista de pipelines).
  2. Na tela de detalhe do pipeline, clique em Nova Tarefa (ou no link/botão equivalente que leva à criação de tarefa com esse pipeline já selecionado).
  3. Preencha o formulário:
    • Pipeline — selecione o pipeline ao qual a tarefa pertence (pode já vir preenchido se você veio do detalhe do pipeline).
    • Nome — nome da tarefa (ex.: "Backup do banco").
    • Descrição — opcional.
    • Tipo de executor — Script (.py, .ps1, …), UiPath ou Blue Prism. Para RPA, veja RPA – UiPath e Blue Prism.
    • Caminho do Script — caminho relativo ao diretório do projeto, ex.: scripts/meu_script.py ou scripts/backup.bat (quando executor = Script).
    • Interpretador — opcional; ex.: /app/venv/bin/python para usar o Python do ambiente do usuário.
    • Tipo — Python (.py), Shell (.sh), Batch (.bat), PowerShell (.ps1), Node.js (.js), Perl (.pl), Ruby (.rb) ou Go (.go). Cada tipo só roda no SO compatível.
    • Depende de — opcional; selecione outras tarefas do mesmo pipeline que precisam ter rodado com sucesso antes desta.
    • Tentativas, Delay entre tentativas, Timeout (s) — retry e tempo máximo.
    • Configuração do Agendamento (JSON) — quando a tarefa roda (veja as tabelas abaixo).
    • Tarefa ativa — marque para a tarefa ser agendada.
  4. Clique em Salvar.

O tipo de agendamento (intervalo, diário, semanal, etc.) é inferido automaticamente a partir das chaves do JSON.

4. Configuração do Agendamento (JSON)

O campo Configuração do Agendamento é um único objeto JSON. O sistema define o tipo de agendamento pelas chaves presentes, conforme a tabela de inferência abaixo.

4.1 Inferência do tipo

Se o JSON contiver...Tipo inferido
seconds, minutes, hours ou daysIntervalo
weekQuinzenal
day (dia do mês, 1–31)Mensal
day_of_week (0–6)Semanal
year, month ou secondCron
Nenhuma das acima (ou só hour/minute)Diário
Vazio {}Diário (meia-noite)

4.2 Intervalo

Ex.: {"minutes": 30} — a cada 30 minutos.

4.3 Diário

Ex.: {"hour": 9, "minute": 0} — todo dia às 09:00. {} — meia-noite.

4.4 Semanal

Ex.: {"day_of_week": 0, "hour": 9, "minute": 0} — segunda às 09:00 (0=segunda, 6=domingo).

4.5 Mensal

Ex.: {"day": 1, "hour": 9, "minute": 0} — dia 1 do mês às 09:00.
